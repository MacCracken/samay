# Getting started with samay

## Build / test / bench

```sh
cyrius deps                              # resolve stdlib + ai-hwaccel into lib/
cyrius build src/main.cyr build/samay    # compile the demo
./build/samay                            # run it
cyrius test  tests/samay.tcyr            # run the test suite (130/130)
cyrius bench tests/samay.bcyr            # benchmarks
```

## Layout

samay is a **library** exposed as a single-file bundle:

- `src/*.cyr` — domain modules (`uuid`, `types`, `scheduler`, `cronexpr`, `cron`,
  `training`), in dependency order.
- `src/lib.cyr` — include-only aggregation header (the whole-library entry).
- `src/main.cyr` — in-tree demo entry (excluded from the bundle).
- `dist/samay.cyr` — the committed bundle consumers include (regenerated by
  `cyrius distlib` from the `[lib].modules` list in `cyrius.cyml`).
- `tests/samay.tcyr` — tests · `tests/samay.bcyr` — benchmarks.
- `rust-old/` — the original Rust source, the parity oracle. **Do not modify.**

## Adding a feature

1. Edit the relevant `src/<module>.cyr` (or add a new module).
2. If new, register it in `cyrius.cyml` `[lib].modules` **and** `src/lib.cyr`,
   in dependency order.
3. Cross-check behavior against `rust-old/` (diverge only with an ADR —
   see [`../adr/template.md`](../adr/template.md)).
4. Add tests to `tests/samay.tcyr`; run `cyrius test tests/samay.tcyr`.
5. Run the cleanliness gate: `cyrius fmt --check`, `cyrius lint`, `cyrius bench`,
   then `cyrius distlib` to refresh `dist/samay.cyr`.
6. Bump `VERSION`, add a CHANGELOG entry, and update `docs/development/state.md`
   before tagging.
